# 1. 🧠 PRODUCT OVERVIEW

## 1.1 Objective

Build a platform where:

* Businesses can create a **mini website**
* Users can **discover businesses**
* Businesses receive **qualified leads**

---

## 1.2 Success Criteria

* A business can be listed in < 5 minutes
* A user can find a business in < 10 seconds
* Lead submission success rate > 95%

* Claim business

### Validation Rules:

  * Claiming business

## 2.3 Admin Roles

### Super Admin:

* Full control

### Co-Admin:

* Same as admin except:

  * Cannot modify/delete super admin

# 3. 🧩 BUSINESS LISTING (CORE FEATURE)

## 3.1 Business Data Model (Functional)

### Required Fields:

* Business Name (string, required, min 3 chars)
* Category (required)
* City (required)
* Phone (required, numeric validation)

### Optional:

* Email
* Website
* Description
* Services
* Products
* Gallery
* FAQs

---

## 3.2 Business Lifecycle

```text
Draft → Submitted → Pending → Approved → Live
                         → Rejected
```

---

## 3.3 Rules

* Only **Approved listings visible publicly**
* Rejected listings:

  * Editable
  * Not visible

---

## 3.4 QA Acceptance

* Submit listing → status must be “Pending”
* Approved listing → visible on search page
* Rejected listing → not visible publicly

# 15. 📘 USER STORIES (DETAILED + QA READY)

---

## 🧩 15.1 BUSINESS LISTING

### US-01: Create Business Listing (Guest)

**As a** guest user
**I want to** submit a business listing
**So that** my business can be visible on the platform

#### Acceptance Criteria:

* User can access submission form without login
* All required fields must be validated before submission
* On submit → listing status = “Pending”
* Listing is NOT visible publicly before approval
* User cannot edit listing after submission

---

### US-02: Create Business Listing (Logged-in)

**As a** logged-in user
**I want to** create and manage my listing
**So that** I can update my business anytime

#### Acceptance Criteria:

* User can save draft
* User can edit listing after submission
* Listing appears under “My Businesses”
* Ownership is assigned to user

---

---

## 🧩 15.2 SEARCH & DISCOVERY

### US-03: Search Business

**As a** user
**I want to** search businesses by keyword and location
**So that** I can find relevant services

#### Acceptance Criteria:

* Search returns results based on:

  * Name, services, products, description
* Results filtered by selected location
* Search query reflected in URL
* Empty state shown when no results

---

### US-04: Apply Filters

**As a** user
**I want to** filter search results
**So that** I can refine results

#### Acceptance Criteria:

* Filters apply using AND logic
* Multiple filters can be selected
* Results update dynamically
* Reset clears all filters
